Project Report: SNMP GetRequest Implementation 
1. Introduction to SNMP 
The Simple Network Management Protocol (SNMP) is an Internet Standard protocol for 
collecting, organizing, and modifying information about managed devices on IP networks. It 
is widely used in network management systems to monitor network-attached devices for 
conditions that warrant administrative attention. SNMP consists of an SNMP manager, an 
SNMP agent, and a Management Information Base (MIB). 
 
2. Explanation of SNMP GET Request 
An SNMP GET request is a message sent by the SNMP manager to the SNMP agent to 
retrieve the value of a specific variable or a set of variables. The agent responds with an 
SNMP GET response message containing the requested data. It is a fundamental operation 
used for monitoring the health and status of network devices without modifying their 
configuration. 
 
3. Explanation of OIDs Used 
Object Identifiers (OIDs) are used to uniquely identify managed objects in an MIB hierarchy. 
In this project, we used two standard OIDs from the `SNMPv2-MIB`: 
- **`1.3.6.1.2.1.1.5.0` (sysName.0):** This OID points to the administratively assigned 
name for the managed node. By convention, this is the node's fully-qualified domain name. 
- **`1.3.6.1.2.1.1.3.0` (sysUpTime.0):** This OID represents the time (in hundredths of a 
second) since the network management portion of the system was last re-initialized. 
 
4. Explanation of PySNMP 
`pysnmp` is a pure-Python, cross-platform SNMP engine implementation. It supports SNMP 
v1/v2c/v3 and provides both high-level and low-level APIs. In this project, we utilized the 
modern, asynchronous High-Level API (`pysnmp.hlapi.asyncio`) which leverages Python's 
`asyncio` for efficient, non-blocking network I/O operations to send our GET requests.
